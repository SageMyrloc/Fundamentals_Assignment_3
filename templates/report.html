Question 2.2: What are the security issues of Web Socket, and what can be done to secure Web Socket? You need to relate this to the security features you have implemented in Task 1.2.

There are plenty of security issues with sockets that are not properly implemented, some of these, according to Lee (2024)include:

  1. Unauthorised access to the socket connection - " A big flaw of WebSocket protocols is that they don't handle authorization/authentication." (Kovacic 2021)
  2. DOS & DDOS attacks - " Denial of Service (DoS) attack exploits vulnerabilities in your WebSocket implementation to crash the WebSocket server, making it inaccessible for your genuine users. " (Booker 2023)
  3. Tunneling - This is a method of transporting data by encapsulating packets within othyer packets. However allowing tunneling leads to potential security breaches through cross-site scripting attacks
  4. Sniffing Attacks - When data is transfered using sockets it is done so in an unencrypted manner by default (Similar to how HTTP works). However this is subject to attacks such as Man-in-the-middle attacks which can steal data being passed between client and server.
  5. Data Masking - Lee (2024) tells that websockets uses data masking to avoid problems such as contaminated browser cache. However, this can be a problem because it can prevent monitoring programmes deteching iteraction patterns and technology to prevent data loss does not even recognise websockets and are unable to monitor their flow.
  6. Vulnerability to Input Data Attacks - "take place when an attacker purposefully enters information into a system or application with the intentions to break the system's functionality. (Fitzgibbons 2019) This type of attack can also be used for instance as SQL injection attacks to subvert a database and gain access to sensitive information.
  7. No Authentication During the Handshake Process - The websocket protocol does not allow end--user authentication during the handhake process. Only typical HTTP methods are accessible. Because of this "HTTP can send validation data directly to the WS." (Lee 2024). This can benefit attackers using methods such as cross-site websocket Abducting. This is where a malicious user creates a website on their own domain that creates a connection to the WS connection that allows two-way interaction with the application.
  8. Unencrupted TCP Channels - Websockets by default are unencrypted, which can lead to issues. (see point 4).
